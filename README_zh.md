**ACLIx** 是一个命令行界面助手，它利用大型语言模型在终端环境中执行任务并回答查询。它作为一个自主智能体运行，能够与主机操作系统、本地文件系统和外部网络服务进行交互。

## 功能特性

- **问答：** 使用本地文件上下文或实时网络搜索来处理用户查询。
- **代码编写与编辑：** 读取、写入和修改本地文件。文件编辑采用精确的字符串替换机制，而不是完全重写文件。
- **命令执行：** 直接在主机上运行 Shell 命令和 Python 脚本。
- **安全提示：** 利用基于 AST（抽象语法树）的评估器来评估 Shell 命令的风险。在运行修改系统状态或带来安全风险的操作之前，会暂停执行以请求用户确认。

## 核心设计

### 智能体编排

系统采用主编排器（Master Orchestrator）模式结合 **Re-Act** 方法。对于复杂的任务，主智能体会将子目标委派给专门的、动态加载的**子智能体（Subagents）**（例如：计划者、探索者、执行者）。子智能体在具有明确的只读或读写权限的隔离上下文中运行，这防止了冲突操作并能有效地管理 Token 限制。

### 工具

工具是提供给智能体以与环境交互的内置原生函数：

- `shell`: 执行操作系统命令。
- `python`: 运行 Python 脚本或内联代码。
- `file_read`: 读取具有分页限制的文件内容。
- `file_write`: 覆盖或创建新文件。
- `file_edit`: 使用忽略空格的精确字符串替换机制来修改现有文件。
- `glob`: 根据命名模式查找文件，自动忽略大型目录。
- `grep`: 在文件内搜索文本。
- `ask_user`: 请求用户输入密码或缺失的参数。
- `web_search`: 使用 Tavily API 在网络上搜索最新信息。
- `read_skill`: 加载特定工作流或标准操作程序 (SOP) 的指令。

### 分层记忆系统

ACLIx 通过多层记忆架构来管理上下文：

- **长期记忆 (LTM):** 在用户级别 (`~/.aclix/ACLI.md`) 和项目级别 (`./ACLI.md`) 的 Markdown 文件中维护的持久状态，规定了永久的事实、约束和偏好。
- **短期记忆 (STM):** 由本地 SQLite 数据库支持的基于会话的对话历史记录，允许跨 REPL 会话恢复状态。
- **压缩记忆 (CM):** 一个自动滚动的摘要系统。当 Token 限制或消息数量超过安全阈值时，引擎会调用 LLM 将历史上下文压缩为密集的摘要，丢弃较旧的原始消息以节省 Token。
- **BM25 检索:** 正在从完整的 LTM 注入过渡到基于本地 BM25 的检索方法。系统将选择性地检索并仅注入每个查询最相关的 3 个记忆片段，以进一步优化 Token 使用和上下文相关性。

### 可扩展的技能与规则

系统架构高度模块化，依赖于基于文件系统的插件模型。

- **技能 (Skills):** 由 `SKILL.md` 文件（以及可选脚本）定义。它们代表了智能体可以学习并执行的可插拔 SOP。
- **规则 (Rules):** 由 `RULE.md` 文件定义。它们会根据当前工作目录将特定的行为约束动态注入到智能体的系统提示词中。
- **渐进式披露:** 该架构利用了渐进式披露技术。智能体不会在初始系统提示词中充斥所有可用的指令，而只会在用户当前上下文或明确意图触发时，才加载详细的 SOP 或特定的规则集。

### 安全性与人机协同 (HITL)

内置的安全评估器将 Shell 命令解析为**抽象语法树 (AST)**，以分配风险级别（`low`（低）、`medium`（中）、`high`（高））。它旨在拦截破坏性命令（例如 `rm -rf /`、`sed -i`）、权限提升和 Fork 炸弹。这实施了严格的**人机协同 (HITL)** 架构，确保在执行任何中高风险操作之前具有强制性的 UI 确认循环。

## 使用方法

### 安装

需要 Node.js v23 或更高版本。

```bash
npm install -g @aliasjeff/acli
```

### 执行模式

- **命令模式:** 通过 `acli chat "<查询内容>"` 触发。它处理单个用户请求，执行任务，并在完成后退出进程。
- **REPL 模式:** 通过运行不带参数的 `acli` 触发。它启动一个交互式 Shell 会话。用户可以保持对话上下文并使用斜杠命令（例如 `/history`, `/compact`, `/rules`, `/skills`, `/config`, `/clear`, `/exit`）。

### 命令

**初始化:**

设置 LLM 提供商，选择模型，并输入 API 密钥。

```bash
acli onboard
```

**交互式会话 (REPL 模式):**

启动保留上下文的持续对话。

```bash
acli
```

**单个任务 (命令模式):**

执行单个目标并退出。

```bash
acli chat "创建一个包含当前日期的文本文件"
```

**配置管理:**

检查或更新当前的配置参数。

```bash
acli config
```
