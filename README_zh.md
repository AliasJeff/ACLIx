# ACLIx

ACLIx 是一个命令行界面助手，利用大语言模型在终端环境中执行任务并回答用户查询。它作为一个自治代理运行，能够与主机操作系统、本地文件系统以及外部网络服务进行交互。

### 功能特性

- **代码编写与编辑:** 读取、写入和修改本地文件。文件编辑采用精确字符串替换机制，而非重写整个文件。
- **问答:** 基于本地文件上下文或实时网络搜索处理用户查询。
- **命令执行:** 直接在宿主机器上运行 Shell 命令和 Python 脚本。
- **安全提示:** 使用基于 AST 的评估器来评估 Shell 命令风险。在执行修改系统状态或具有安全风险的操作前，暂停执行并请求用户确认。

### 核心概念

#### Agent 与 Re-Act

系统使用基于 Re-Act (推理与行动) 模式的代理工作流。对于给定任务，代理生成推理步骤以确定所需操作，执行一个或多个工具，并评估输出结果。该循环会迭代进行，直到达成目标或达到最大步数限制。

#### Tools

工具是提供给代理以与环境交互的原生函数：

- `shell`: 执行操作系统命令。
- `python`: 运行 Python 脚本或内联代码。
- `file_read`: 带有分页限制读取文件内容。
- `file_write`: 覆盖或创建新文件。
- `file_edit`: 使用精确字符串替换修改现有文件。
- `glob`: 基于命名模式查找文件。
- `grep`: 在文件内搜索文本。
- `ask_user`: 向用户请求密码或缺失的参数。
- `web_search`: 使用 Tavily API 在网络上搜索最新信息。
- `read_skill`: 加载特定技能的操作说明。

#### Skills

Skills 是使用 Markdown 格式 (`SKILL.md`) 定义的标准操作程序，作为特定任务的工作流。当任务符合技能描述时，代理使用 `read_skill` 工具动态加载逐步的执行指令。

#### Rules

Rules 是使用 Markdown 格式 (`RULE.md`) 定义的系统约束。这些规则被注入到代理的系统提示词中以强制执行行为策略，如语言要求或编码规范。

#### 记忆与压缩

- **记忆:** 会话历史使用 SQLite 数据库 (`acli.db`) 在本地保存。上下文以当前工作目录为键，允许在同一目录重启 REPL 模式时自动恢复会话。
- **记忆压缩:** 系统使用分词器监控 Token 使用情况。当上下文超过 80,000 tokens 时，后台会运行压缩进程。该进程截断过长的工具输出，并使用语言模型生成早期交互片段的语义摘要。该摘要替换原始消息，从而在保持上下文的同时减少 Token 数量。用户也可以通过 `/compact` 命令手动触发此过程。

#### 可扩展性

系统支持在不修改源代码的情况下进行扩展。用户可以通过创建 `.aclix/skills` 或 `.aclix/rules` 目录来定义自定义技能和规则。这些目录可以放置在用户主目录中以实现全局作用域生效，或放置在当前工作目录中以实现项目级作用域生效。项目级配置会覆盖用户级配置。

### 使用方法

#### 使用 npm 安装 (node v23)

`npm install -g @aliasjeff/acli`

#### 运行模式

- **命令模式:** 通过 `acli chat "<query>"` 触发。系统处理单一用户请求，执行任务并在完成后退出进程。
- **REPL 模式:** 无参数运行 `acli` 触发。系统启动一个交互式 Shell 会话。用户可以保持对话上下文，并使用斜杠命令 (例如 `/history`, `/compact`, `/rules`, `/skills`, `/config`, `/clear`, `/exit`)。

#### 常用命令

**初始化:**

设置语言模型供应商、选择模型并输入 API 密钥。

```
acli onboard
```

**单次任务 (命令模式):**

执行单一目标并退出。

```
acli chat "Create a text file containing the current date"
```

**交互式会话 (REPL 模式):**

启动保留上下文的持续对话。

```
acli
```

**配置管理:**

查看或更新当前配置参数。

```
acli config
```

### 架构与目录结构

项目被划分为多个独立模块以分离关注点：

- **CLI 层:** 解析命令行参数，处理用户输入、进程中断信号以及终端渲染。
- **Core 层:** 包含代理执行逻辑、提示词构建、安全评估、Token 管理以及工具定义。
- **REPL 层:** 管理交互式会话状态和斜杠命令路由。
- **Services 层:** 聚合外部依赖，包括用于数据库存储的 SQLite、本地配置管理、宿主进程执行以及通过 Vercel AI SDK 调用的语言模型提供商。

```
.
├── bin/                 # CLI 入口文件
├── src/
│   ├── cli/             # 命令行解析、中间件与中断处理
│   ├── core/            # 核心系统逻辑
│   │   ├── agent/       # 聊天工作流与提示词生成
│   │   ├── context/     # 运行时上下文追踪
│   │   ├── memory/      # Token 计数与上下文压缩
│   │   ├── rules/       # 规则管理器
│   │   ├── security/    # Shell 命令风险评估器
│   │   ├── skills/      # 技能管理器
│   │   └── tools/       # 工具定义
│   ├── repl/            # 交互式会话引擎与斜杠命令注册表
│   ├── services/        # 配置、数据库、执行器、模型接口与日志
│   ├── shared/          # 类型、错误与常量
│   └── ui/              # 终端提示与视觉组件
└── package.json
```
